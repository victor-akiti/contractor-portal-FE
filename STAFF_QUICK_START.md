# Staff Quick Start Guide

## Welcome to the Amni Staff Portal!

This guide will help you start reviewing and approving contractor registrations quickly.

## Step 1: Getting Access (One-Time Setup)

Your administrator will provide:
- **Email address** (your username)
- **Initial password**
- **Your role** (VRM, CO, HOD, etc.)

### First Login

1. Navigate to **Staff Login** page at `/login/staff`
2. Enter your email and password
3. Click "Login"

**Tip**: Bookmark the staff login page for quick access!

## Step 2: Understanding Your Role

Your role determines what you can approve:

| Role | Primary Responsibility | Approval Stage(s) |
|------|------------------------|-------------------|
| **Admin** | Full system access | All stages |
| **HOD** | Department oversight | A, E |
| **VRM** | Vendor risk management | B |
| **CO** | Contracts review | B, C |

| **Executive Approver** | Executive review | D |
| **Supervisor** | Supervisory oversight | Varies |

**Your role is shown** in the top-right corner when logged in.

## Step 4: Reviewing Applications

### Accessing Pending Applications

1. Click **"Registration Approvals"** in sidebar
2. Go to **"Pending L2"** tab (shows applications awaiting review)
3. Number badge shows count of pending items

### Understanding the Approval Tabs

| Tab | What's Here |
|-----|-------------|
| **Pending L2** | New applications awaiting your review |
| **Completed L2** | Applications that passed initial stages |
| **Returned to Contractor** | Applications sent back for revisions |
| **In Progress** | Applications currently being reviewed |
| **L3** | Escalations requiring higher-level review |
| **Invited Contractors** | Track invitation status |
| **Park Requests** | Applications on hold |

### Reviewing an Application

1. **Find the application**:
   - Scroll through the list
   - Or use search box (company name, email, ID)
   - Or use filters

2. **Click to open** the application

3. **Review all sections**:
   - Company information
   - Documents and certificates
   - Services offered
   - References
   - Financial details

4. **Check for completeness**:
   - All required documents uploaded?
   - Documents clear and legible?
   - Certificates valid (not expired)?
   - Information appears accurate?

## Step 5: Taking Action on Applications

### Option 1: Approve and Move Forward

**When to use**: Application is complete and meets requirements

1. Review all sections carefully
2. Add any remarks/conditions in the comments box (optional)
3. Click **"Approve"** or **"Move to Stage [X]"** button
4. Application advances to next stage

**Tip**: Add remarks to highlight important notes for the next reviewer.

### Option 2: Return to Contractor

**When to use**: Information is missing, incorrect, or needs clarification

1. Identify which sections need attention
2. Click on the section to add **specific remarks**
3. Clearly explain what needs to be changed or added
4. Click **"Return to Contractor"** button
5. Contractor receives notification with your feedback

**Best Practice**: Be specific! Instead of "Documents unclear", say "Business license image is too blurry - please upload a clearer scan."

**Important**: System prevents double-clicking to avoid duplicate actions.

### Option 3: Park Application

**When to use**: You need more time or information before deciding

1. Click **"Park Request"** button
2. Add a reason for parking (e.g., "Waiting for legal review")
3. Application moves to "Park Requests" tab
4. Later, approve or decline the park request

### Option 4: Revert to L2

**When to use**: Application at higher stage needs to go back

1. Click **"Revert to L2"** button
2. Select the specific stage to revert to
3. Add reason for reversion
4. Confirm action

**Note**: Use carefully - reverting disrupts workflow. Consult supervisor if unsure.

   - Contact Person Name

- **Expired**: Past expiry date (can renew these)

1. Go to **"Expired"** tab

## Approval Workflow Overview

The system uses **6 approval stages** (A through F):

| Stage | Level | Description | Typical Role |
|-------|-------|-------------|--------------|
| **A** | 0 | Initial document review | Admin, HOD |
| **B** | 1 | Department/risk review | VRM, CO |
| **C** | 2 | Service verification | CO, GM |
| **D** | 3 | Executive business review | Executive, GM |
| **E** | 4 | Final verification | Admin, HOD |
| **F** | 5 | Completion & activation | Admin |

**Your role** determines which stage(s) you can approve.

Applications flow through stages sequentially. Each stage represents a different level of review and authority.

## Best Practices

### For Efficient Reviews

**Do**:
- Review applications in order received (FIFO)
- Be thorough - check all sections
- Add clear, specific feedback when returning
- Use remarks to communicate with next reviewer
- Document any unusual situations
- Respond to end-user requests promptly

**Don't**:
- Rush through reviews
- Approve without checking all documents
- Give vague feedback ("needs improvement")
- Approve expired certificates
- Skip documentation when parking applications
- Forget to unpark applications

### Communication Tips

When returning applications to contractors:

1. **Be specific**: "Upload company insurance certificate (PDF format)" not "Missing documents"
2. **Be helpful**: Explain what you need and why
3. **Be clear**: Use simple language, avoid jargon
4. **Be complete**: List ALL issues in one return to avoid multiple cycles
5. **Be professional**: Maintain courteous tone

### Certificate Verification

Always check:
- Certificate is uploaded
- Document is clear/legible
- Expiry date is entered correctly
- Certificate is still valid (not expired)
- Certificate type matches requirement

**Red flag**: Certificate expiring soon (within 30 days) - note in remarks.

## Common Scenarios

### Scenario 1: Incomplete Application

**Situation**: Contractor submitted but several documents missing

**Action**:
1. Review what's missing
2. Click sections where documents needed
3. Add remarks listing each missing item
4. Click "Return to Contractor"
5. Application goes back for completion

### Scenario 2: Expired Certificate

**Situation**: Insurance certificate expired last month

**Action**:
1. Note the expiry date in remarks
2. Specify which certificate needs renewal
3. Return to contractor with clear instructions
4. Do NOT approve with expired certificate

### Scenario 3: Unclear Document

**Situation**: Business license scan is blurry/unreadable

**Action**:
1. Add remark: "Business license image too blurry - please upload clearer scan"
2. Return to contractor
3. Wait for updated upload

### Scenario 4: Need Specialist Review

**Situation**: Technical certificate requires specialist verification

**Action**:
1. Assign appropriate end-user(s)
2. Add remarks explaining what to review
3. Wait for end-user completion
4. Review end-user feedback
5. Approve or return based on findings

### Scenario 5: Application Looks Good

**Situation**: All documents present, clear, valid, and accurate

**Action**:
1. Complete your thorough review
2. Add any remarks for next stage (optional)
3. Click "Approve"
4. Application moves to next stage

## Troubleshooting

### Can't see pending applications?
- Verify you're on the correct tab
- Check if filters are applied (clear them)
- Refresh the page
- Verify your role has approval permissions

### Action button disabled/grayed out?
- You may not have permission for that stage
- Application may be at different stage
- Another user may be reviewing it
- Try refreshing the page

### Can't assign end-users?
- Feature may not be available at current stage
- Verify you have permission
- Check if end-users exist in system
- Contact administrator

### Application disappeared?
- It may have moved to another tab (check all tabs)
- Another user may have acted on it
- Check Events log to see what happened
- Use search to locate it

## Frequently Asked Questions

**Q: How many applications should I review per day?**
A: Varies by role and complexity. Focus on quality over speed. Typical: 5-15 applications per day depending on complexity.

**Q: What if I'm unsure about approving?**
A: Park the application and consult your supervisor. Better to ask than approve incorrectly.

**Q: Can I un-return an application?**
A: No, but an administrator can use "Revert to L2" if needed. Be careful before clicking "Return".

**Q: What if contractor hasn't responded to my return?**
A: Check how long it's been. Send reminder via messaging after 5-7 days. Contact through invitation email if no response after 14 days.

**Q: Can I approve applications outside my stage?**
A: No, system enforces role-based permissions. You can only act on your assigned stage(s).

**Q: How do I know if my delegate is handling my work while I'm OOO?**
A: Check Events log after returning. You'll see actions taken by your delegate.

**Q: Can I see all my approval history?**
A: Yes! Use the Events log and filter by your username.
